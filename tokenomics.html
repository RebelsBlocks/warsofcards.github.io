Tokenomics

Token Contract Address

ft.warsofcards.near

Token Mechanics

The token uses a whitelisted permission architecture that allows approved game contracts to mint rewards and burn bets. This enables seamless integration of games like Blackjack (Live) and Poker (Coming Soon) without changing the core token logic. Once a contract is whitelisted, it automatically gains mint/burn permissions, making Wars of Cards a scalable multi-game token economy.

Token Acquisition

  • Daily free claims
  • Token packs purchases
  • Game winnings (minted by authorized game contracts)

Token Consumption

  • Burning during gameplay
  • Users cannot burn tokens directly, only games can burn

Supply Model

Cards token features an unlimited supply model with dynamic circulation, where supply is naturally regulated by game mechanics rather than fixed caps. This supply-agnostic design allows the ecosystem to scale across multiple games without artificial constraints, supporting unlimited growth of the Wars of Cards World.

Supply Calculation

Circulating supply = total_minted − total_burned

Tracked Values

  • total_supply — current total supply
  • total_minted — cumulative tokens minted
  • total_burned — cumulative tokens burned

Treasury

The Treasury operates as the central revenue collection mechanism and increases with player activity.

Revenue Sources

  1. Transfer fees — 7% on non-whitelisted token transfers (tracked via total_fees_collected)
  2. Pack purchases — 100% of NEAR deposits from CARDS Pack sales

Treasury NEAR account: treasury.warsofcards.near
